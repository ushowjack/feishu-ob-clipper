# 财摘开发约束

本文件是项目级强制开发规则。修改代码前必须先阅读并遵守。

## 一类网站，一个独立爬虫模块

- 每新增一类网站或域名，必须新增一个独立的 `src/<site>-site.js` 爬虫模块。
- 禁止把新网站的选择器、正文定位、动态加载、页面清理或图片特例继续堆进现有爬虫文件、`content.js` 或 `content-core.js`。
- 禁止仅通过在公共文件中增加一串域名 `if/else` 来实现新网站抓取。
- 修改现有网站的抓取规则时，只修改该网站自己的爬虫模块及其直接相关测试。

当前模块边界：

- `src/feishu-site.js`：飞书正文识别、虚拟滚动采集、飞书页面清理和临时图片处理。
- `src/scys-site.js`：生财正文识别、正文克隆、独立图片列表和生财图片请求规则。
- `src/site-rules.js`：网址识别和网站类型映射，不承载具体正文抓取逻辑。
- `src/content-core.js`：网站路由和真正被多个爬虫复用的通用能力。
- `src/content.js`：Chrome 消息入口和调用对应爬虫，不写网站专属选择器。
- `src/markdown.js`、`src/vault.js`：所有网站共用的 Markdown 转换和本地保存能力。

## 新增网站的完成清单

新增一个网站时，至少完成以下内容：

1. 新建 `src/<site>-site.js`，封装该网站完整的正文抓取流程。
2. 在 `src/site-rules.js` 增加网址识别；只在公共路由处注册新爬虫。
3. 按需更新 `manifest.json` 的页面匹配、域名权限和可访问模块。
4. 为新爬虫增加独立测试，覆盖正文根节点、全文完整性、图片识别和非正文排除。
5. 使用真实文章验证正文首尾、图片数量和图片实际下载结果。
6. 运行 `npm test`、`npm run check` 和 `npm run package`。

## 公共代码的准入条件

- 只有已经被两个或以上爬虫真实复用的逻辑，才可以放入公共模块。
- 某个网站独有的 DOM 类名、选择器、接口地址、加载方式和容错规则必须留在该网站模块。
- 不为尚未接入的网站提前设计抽象层；优先保持每个爬虫独立、可测试、可替换。
- 代码评审发现网站专属逻辑写入公共文件时，视为未完成，必须先拆回独立爬虫模块。

## 修改范围与验证

- 只修改当前网站爬虫和必要的公共注册点，不顺带重写其他网站逻辑。
- 修复抓取问题时先用真实页面确认根因，再增加回归测试，最后修改代码。
- 测试通过不等于抓取成功；涉及正文或图片的改动必须补一次真实页面验收。
