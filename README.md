# 飞书正文保存到 Obsidian

一个无需飞书开放平台应用的 Chrome 插件。它读取当前已打开的飞书 Wiki/文档，把正文转换为 Markdown，并将图片保存到你授权的 Obsidian Vault。

## 功能

- 使用当前 Chrome 的飞书登录状态，支持有权限访问的 `wiki`、`docx` 和 `docs` 页面。
- 弹窗采用 Obsidian 风格的属性编辑界面，保存前可修改当前文章属性。
- 默认模板可新增任意字段，并支持文本、多值列表、日期、布尔值和数字。
- 模板保存在浏览器本地；单篇文章的临时修改不会改变模板默认值。
- 保存标题、段落、标题层级、列表、任务列表、引用、代码块、表格、链接、分隔线和图片。
- Markdown 属性完全由当前模板与保存前编辑值生成，不强制添加隐藏字段。
- 图片保存到可配置的 Vault 附件目录，正文使用 Obsidian `![[...]]` 链接。
- 同名笔记自动生成 `标题-2.md`、`标题-3.md`，不会覆盖旧笔记；附件同样不会覆盖旧文件。
- 文档内容只在浏览器和本地 Vault 之间处理，不上传到第三方服务。

## 安装

建议使用最新版桌面版 Google Chrome。

1. 在地址栏打开 `chrome://extensions`。
2. 打开右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本项目文件夹（文件夹内应能看到 `manifest.json`）。
5. 建议把插件固定到 Chrome 工具栏。

修改插件代码后，需要在 `chrome://extensions` 中点击该插件的“重新加载”，并刷新已经打开的飞书页面。

## 使用

1. 在 Chrome 中登录飞书，打开一个你有权限访问的 Wiki 或文档页面，等待正文显示完成。
2. 点击工具栏中的“飞书正文保存到 Obsidian”。
3. 首次使用点击“选择 Obsidian Vault”，在系统目录选择器中选择 Vault 根目录并允许写入。
4. 展开“保存设置”：
   - “笔记目录”留空时写入 Vault 根目录，也可填写 `raw/01-articles` 之类的相对路径。
   - “附件目录”默认为 `attachments/feishu`。
5. 点击“保存到 Obsidian”。成功后弹窗会显示写入的相对路径和图片数量。

目录不存在时会自动创建。路径只能相对于 Vault，`/绝对路径`、`C:\路径` 和含 `..` 的路径会被拒绝。

飞书长文采用虚拟渲染，只把当前位置附近的正文放进页面。保存长文时，插件会短暂遍历文档滚动区、按飞书块 ID 收集完整正文，然后恢复原滚动位置；此过程通常需要数秒。

## 常见问题

### 提示“插件尚未注入当前页面”

刚安装或重新加载插件后，刷新当前飞书页面再试。Chrome 不会自动把新版本内容脚本注入安装前已打开的标签页。

### 提示“没有识别到已加载的飞书正文”

确认页面正文已经显示、当前账号有权限并等待加载完成。如果飞书刚更新页面结构，也可能需要更新插件的正文识别规则。

### 保存长文时页面为什么会滚动

这是为了读取飞书按滚动位置动态加载的正文块。插件完成采集后会恢复原位置，不会修改飞书文档。

### 再次打开 Chrome 后提示恢复 Vault 授权

浏览器可能要求重新确认以前选择的目录。点击“恢复 Vault 授权”；如果仍失败，展开设置并“重新选择 Vault”。

### 部分图片没有保存

飞书图片可能使用临时地址、跨域限制或已过期签名。插件仍会保存正文，并保留远程图片链接，同时在结果中提示失败数量。

### 为什么没有评论、画板或电子表格内容

第一版只处理已渲染的普通文档正文，不包含评论、修订历史、画板、数据库视图、电子表格以及复杂第三方嵌入。

## 开发验证

项目没有运行时依赖或构建步骤。需要 Node.js 20 或更高版本运行测试：


## 权限说明

- `activeTab`：读取用户主动点击插件时的当前标签页信息。
- `storage`：保存笔记目录和附件目录设置。
- `https://*.feishu.cn/*`：仅在飞书页面注入正文读取脚本。
- Vault 目录授权由 Chrome 的系统目录选择器单独请求；目录句柄保存在插件自己的 IndexedDB 中，不会发送给飞书页面。
