# 飞摘公开 README 设计

## 目标

为第一次接触“飞摘”的普通用户提供一份可快速理解、安装并完成首次保存的中文说明，同时保留维护者需要的验证与发布入口。README 与当前插件名称、界面和公开 Release 流程保持一致。

## 读者与成功标准

- 主要读者：从作者内容渠道来到 GitHub、会使用 Chrome 和 Obsidian、但不熟悉源码安装的用户。
- 首要任务：下载 Release、加载插件、选择 Vault、把一篇飞书文档保存到 Obsidian。
- 成功标准：用户只阅读 README 即可完成上述流程；截图在 GitHub 正常显示；隐私边界、支持范围和常见失败有明确说明。

## 内容结构

1. 品牌区：Logo、产品名、一句话价值、适用环境和隐私承诺。
2. 效果预览：展示插件弹窗与保存结果，帮助用户先确认产品用途。
3. 四步上手：下载、Chrome 安装、首次授权、保存文档。
4. 功能与支持范围：说明可保存的正文结构、图片、属性和防覆盖行为。
5. 常见问题：只保留安装、注入、正文加载、Vault 授权、长文滚动和图片失败等高频问题。
6. 安全与权限：用普通用户能理解的语言解释数据流和 Chrome 权限。
7. 维护者区域：折叠展示源码安装、测试、打包和发布命令。

## 配图方案

- 使用真实插件界面，不制作与实际产品不一致的概念图。
- 配图保存于 `docs/images/`，README 使用仓库相对路径，保证 GitHub 长期可访问。
- 至少包含一张完整弹窗图和一张四步安装/使用说明图；若本机实际浏览器状态允许，再补保存成功状态图。
- 图片文字使用中文，避免依赖外部图床，不展示真实 Vault 路径、账号、文档内容或其他敏感信息。
- 图片宽度兼顾桌面和移动端 README 阅读，关键操作同时保留文字步骤，避免只靠截图传达信息。

## 发布边界

- 用户已授权将当前工作区的现有插件改动一起公开。
- 提交前检查全部改动，排除缓存、构建产物、账号信息、绝对路径和密钥。
- 运行测试、语法检查和 Release 打包；失败则不推送并报告具体原因。
- 目标为公开仓库 `ushowjack/feishu-ob-clipper`，远程地址必须使用 SSH。
- 推送 `main` 后重新读取 GitHub 仓库状态，核验 README 与图片已经上线。

## 非目标

- 本次不新增插件功能，不调整既定产品交互，不发布 Chrome Web Store。
- 不承诺飞书评论、画板、电子表格、数据库视图或复杂嵌入内容的导出。
- 不使用第三方统计、外部图片托管或付费服务。

## 验证清单

- README 的下载链接、图片路径、文内链接和命令均可解析。
- README 与 `manifest.json`、`package.json`、实际弹窗文案一致。
- `npm test`、`npm run check`、`npm run package` 成功。
- `git diff --check` 无空白错误，提交不含敏感信息或无关生成物。
- Git 远程为 SSH，GitHub 默认分支可看到 README 和全部配图。
