# 飞书正文保存到 Obsidian Chrome 插件设计

## 目标

开发一个无需飞书开放平台应用的 Chrome Manifest V3 插件。用户在已登录且有权访问的飞书 Wiki/文档页面点击插件，即可把当前正文转换成 Markdown，连同图片保存到用户授权的 Obsidian Vault。

完成必须满足：

- 支持 `*.feishu.cn/wiki/*` 和常见飞书文档页。
- 首次使用选择一次 Vault；后续复用授权，失效时明确提示重新授权。
- 笔记目录和附件目录可配置，目录不存在时自动创建。
- Markdown 文件按飞书标题命名，清理非法字符；同名时使用 `标题-2.md` 等新文件，绝不静默覆盖。
- 保存标题、段落、标题层级、列表、引用、代码块、表格、链接、分隔线和图片。
- 图片写入 Vault 附件目录，正文使用 Obsidian `![[...]]` 链接。
- 文件带 YAML 元数据：标题、来源 URL、抓取时间、来源类型。
- 页面未登录、无权限、正文未加载、正文识别失败、目录授权失效和图片失败都有可理解的提示。

## 非目标

- 不调用飞书开放平台 API，不要求企业管理员授权。
- 第一版不处理飞书评论、修订记录、协作者、数据库视图和画板内容。
- 第一版不保证还原复杂嵌套布局、电子表格或第三方嵌入的视觉样式。
- 不自动覆盖或更新以前保存的同名笔记。

## 架构

插件由四个独立部分组成：

1. **弹窗界面**：显示当前页面状态、目标路径、保存按钮和结果；只负责交互编排。
2. **内容脚本**：在当前飞书标签页定位已渲染正文、克隆并清理页面节点，输出中立的文章数据。
3. **Markdown 转换器**：把清理后的 HTML 转成 Markdown，并产生待下载图片清单；不依赖 Chrome API，便于单元测试。
4. **Vault 写入器**：通过 File System Access API 管理目录授权、创建子目录、写图片与 Markdown，并处理同名递增。

目录句柄存入扩展自身的 IndexedDB；普通设置存入 `chrome.storage.local`。目录句柄不发送到网页，内容脚本也不能访问本地文件系统。

## 数据流

1. 用户打开有权限的飞书文档，等待正文显示后点击插件。
2. 弹窗检查 URL 和 Vault 授权；若未选择 Vault，用户点击按钮触发目录选择器。
3. 弹窗向内容脚本请求文章数据。内容脚本按多组稳定特征查找正文，排除导航、工具栏、评论和隐藏节点。
4. 转换器生成 Markdown 和图片清单。
5. 写入器逐张获取图片。优先由内容脚本在飞书页面上下文使用现有登录态获取二进制；失败则在 Markdown 中保留原始 URL，并记录警告。
6. 写入器先保存成功获取的图片，再以不覆盖策略写 Markdown。
7. 弹窗显示最终文件相对路径、图片成功/失败数量和警告。

## 页面识别策略

正文识别采用候选选择器加评分，不只依赖单一 class：

- 优先查找编辑器、文档内容区、ProseMirror/可编辑根节点等语义特征。
- 候选按可见文本长度、块级元素数量、标题/段落密度评分。
- 排除明显的导航栏、目录、评论区、工具栏和不可见区域。
- 结果文本过短或正文节点为空时停止保存，提示用户确认正文已加载。

页面结构适配集中在一个模块，飞书改版时无需修改 Markdown 与文件写入逻辑。

## Markdown 与图片规则

- 文档标题写为一级标题；正文原有一级标题下移一级，避免重复主标题。
- 支持段落、`h1`–`h6`、有序/无序/任务列表、粗体、斜体、删除线、行内代码、代码块、引用、链接、表格和分隔线。
- 连续空行压缩为最多两个换行；保留代码块内部空白。
- YAML 字符串安全转义，日期使用带时区的 ISO 8601 格式。
- 图片名使用 `笔记标题-01.ext`，按出现顺序编号；扩展名优先取响应 MIME 类型，再取 URL，最终回退为 `png`。
- 图片引用使用相对 Vault 的 Obsidian 内链。图片获取失败时使用标准 Markdown 外链，并在保存结果中提示。

## 路径与安全

- 设置项只接受 Vault 内相对目录，不接受绝对路径和 `..` 路径段。
- 所有目录通过已授权 Vault 句柄逐级创建，无法越出 Vault。
- 文件名移除操作系统非法字符、控制字符、结尾点和空格，并限制长度；空标题回退为“未命名飞书文档”。
- 默认笔记目录为 Vault 根目录，默认附件目录为 `attachments/feishu`。
- 不上传文档内容，不引入远程脚本，不使用第三方服务。

## 错误处理

- 非飞书页面：禁用保存并提示支持的 URL。
- 未登录或无权限：识别登录/错误页面，提示先在当前标签页获得访问权限。
- 正文未加载：不给出成功结果，也不创建空笔记。
- 授权丢失：保留设置，提示重新选择 Vault。
- 图片部分失败：正文仍保存，失败图片保留远程链接，返回部分成功状态。
- Markdown 写入失败：报告失败，不宣称保存完成；已写图片允许保留，不删除用户文件。
- 同名文件：自动尝试 `-2`、`-3`，设置合理上限，达到上限时提示用户调整标题或目录。

## 界面

弹窗保持单屏：顶部显示“飞书 → Obsidian”和当前文档标题；中部显示 Vault、笔记目录、附件目录；底部是主按钮“保存到 Obsidian”。首次未授权时主按钮为“选择 Obsidian Vault”。保存中禁用重复点击，成功后显示相对路径，错误信息给出直接可执行的解决方法。

设置直接放在弹窗的折叠区域，不另做复杂设置页。第一版使用原生 HTML/CSS/JavaScript，不引入构建工具和运行时依赖，便于在 Chrome 的“加载已解压扩展程序”中直接安装。

## 验证

自动化测试覆盖纯逻辑：

- 非法文件名与空标题清理。
- 相对路径校验与越界拒绝。
- 同名文件递增规则。
- HTML 到 Markdown 的标题、列表、代码、表格、链接与图片转换。
- YAML 转义和图片命名。
- 正文候选评分、隐藏/无关节点排除。

浏览器级人工验证覆盖：

- 在示例形式的飞书 Wiki 页面中提取正文。
- 首次授权、再次打开复用授权、撤销授权后恢复。
- 文件和图片真实写入测试 Vault，并可在 Obsidian 中正常打开。
- 同名保存两次不覆盖。
- 无权限页、未加载页、图片请求失败和非飞书页面提示正确。

若无法访问用户提供的私有页面，则使用本地飞书结构夹具验证转换和写入逻辑，并把真实页面结构适配列为需要用户侧加载插件后确认的风险点，不能据此宣称私有页面已经完全验证。
