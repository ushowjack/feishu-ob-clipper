# 弹窗文字与间距优化设计

## 目标

在不改变现有结构、功能和品牌元素的前提下，参考 Obsidian Web Clipper 的视觉节奏，调整弹窗的文字层级、行高、留白和对齐，使 420px 宽界面更紧凑、清晰、协调。

## 完成标准

- 保留 Logo、飞摘名称、设置入口、属性编辑、正文预览和底部操作区。
- 标题、区块标题、属性名、属性值和辅助信息形成稳定的字号层级。
- 属性行的列宽、行高和间距统一，长标题与长属性值不造成横向溢出。
- 正文预览、状态提示、目录输入框和主按钮之间具有清晰的区隔。
- 主界面与设置页共用一致的字体、圆角和 4/8px 间距节奏。
- 现有自动测试全部通过。

## 视觉标尺

- 基础正文：14px，行高 1.5。
- 主标题：20px，行高 1.42，字重 720。
- 区块标题：14px，行高 1.4，字重 680。
- 属性行：最小高度 42px，属性名与属性值均为 14px。
- 辅助信息：11px 至 12px，行高不低于 1.45。
- 页面水平留白：18px；主要区块垂直间距采用 8px、16px、24px。
- 底部输入框最小高度 40px，主按钮最小高度 48px。

## 改动范围

仅修改 `popup.css` 的排版参数，并在 `tests/popup-ui.test.js` 增加可重复执行的样式约束测试。HTML 结构、JavaScript 行为、数据模型和保存流程均不改动。

## 边界与失败判定

- 420px 宽弹窗出现横向滚动或属性输入列被挤压到不可用，视为失败。
- 长标题无法自动增高、状态文本无法换行或遮挡主按钮，视为失败。
- 设置页因全局字号调整出现字段重叠，视为失败。
- 任一现有测试失败，视为回归。

## 验证

- 自动检查关键字号、间距、最小高度和溢出规则。
- 运行 `npm test` 验证完整测试集。
- 静态检查 CSS 选择器与 420px 宽网格尺寸，确认没有新增横向溢出路径。
