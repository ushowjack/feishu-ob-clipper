# 可配置属性的 Obsidian 风格弹窗设计

## 目标

将当前 Chrome 插件弹窗改造成接近 Obsidian Web Clipper 的深色编辑界面。用户既能在每次保存前临时编辑当前文章属性，也能维护持久化的默认属性模板。保存后的 Markdown frontmatter 必须保留正确的数据类型，并继续兼容现有的飞书正文提取、图片本地化、Vault 授权、目录配置和同名文件防覆盖能力。

## 完成标准

- 弹窗采用深色单栏界面，在约 420px 宽度下无横向溢出，底部保存区始终可操作。
- 默认属性能自动填入文章标题、飞书地址、创建日期和 `clippings` 标签。
- 用户可新增任意属性，并可改名、删除、排序、启用或停用属性。
- 属性支持文本、多值列表、日期、布尔值和数字五种类型。
- 当前文章的临时修改只影响本次保存；模板修改在关闭并重新打开插件后仍然存在。
- 输出 YAML 可被 Obsidian 正确识别为对应类型。
- 原有正文、图片、目录、权限恢复和防覆盖功能不回退。

## 界面结构

弹窗采用约 420px 宽的深色单栏布局，内容高度超过可视区域时纵向滚动。

### 主编辑页

1. 顶部工具栏显示当前模板名 `Default`，右侧提供设置入口。
2. 工具栏下方显示当前文章标题，点击后可直接编辑。
3. 属性区逐行显示类型图标、字段名和字段值。
4. 点击字段值进入对应类型的编辑控件。
5. 字段行菜单提供改名、修改类型和删除操作；属性可调整顺序。
6. 属性区底部提供“添加属性”入口。
7. 正文区显示提取后的纯文本预览，限制最大高度并允许内部滚动；第一版不在弹窗内编辑正文。
8. 底部固定区显示笔记目录和紫色主按钮“添加到 Obsidian”。
9. Vault 未选择或权限失效时，主按钮依次切换为“选择 Obsidian Vault”或“恢复 Vault 授权”。
10. 状态与错误信息显示在底部固定区内，不额外占据大块界面。

### 模板设置页

设置入口在弹窗内部切换到模板设置页，不新开浏览器页面。设置页允许：

- 新增、删除和排序默认属性。
- 修改字段名、显示名称、类型、默认值、自动取值来源和启用状态。
- 配置笔记目录和附件目录。
- 查看当前 Vault 名称并重新选择 Vault。
- 保存模板或取消修改并返回主编辑页。

进入设置页前的当前文章临时修改保留在内存中，返回后继续显示。模板只有点击保存后才写入本地存储。

## 属性模型

每个模板属性使用以下结构：

```js
{
  id: "稳定的本地标识",
  key: "写入 YAML 的字段名",
  label: "界面显示名称",
  type: "text | list | date | boolean | number",
  defaultValue: "与类型匹配的默认值",
  source: "none | title | url | createdDate",
  enabled: true
}
```

- `id` 用于界面更新和排序，不写入 Markdown。
- `key` 在当前模板中必须唯一，保存时按该名称生成 YAML。
- `label` 只影响界面显示；为空时回退为 `key`。
- `defaultValue` 在没有自动取值或自动取值为空时使用。
- `source` 第一版支持文章标题、当前网址和当天日期；选中自动来源后，打开弹窗时解析为当前文章的临时值。
- `enabled` 为 `false` 的字段不进入当前文章属性，也不写入 YAML。

默认模板为：

| 字段 | 类型 | 自动来源或默认值 |
| --- | --- | --- |
| `title` | 文本 | 当前文章标题 |
| `source` | 文本 | 当前飞书地址 |
| `author` | 多值列表 | 空列表 |
| `published` | 日期 | 空值 |
| `created` | 日期 | 当天日期 |
| `description` | 文本 | 空值 |
| `tags` | 多值列表 | `clippings` |

## 编辑与类型规则

- 文本使用单行输入；较长内容仍保存为 YAML 字符串。
- 多值列表允许使用逗号或回车分隔，界面解析后去除首尾空白和空项。
- 日期使用日期输入，值为空或符合 `YYYY-MM-DD`。
- 布尔值使用开关，只能保存为 `true` 或 `false`。
- 数字使用数字输入，允许整数、小数和负数，但不接受非有限值。
- 修改字段类型时尽量转换现有值；无法无损转换时清空该值，并明确提示用户。
- 空文本、空日期和空列表仍保留字段，以便用户进入 Obsidian 后继续填写。
- 字段顺序等于生成 YAML 时的顺序。

字段名去除首尾空白后不得为空，不得重复，也不得包含换行或其他控制字符。普通 YAML 特殊字符允许存在，由序列化器负责安全引用。

## 数据流

1. 弹窗启动时并行读取当前标签页、Vault 目录句柄、保存目录设置和属性模板。
2. 使用模板自动来源和默认值生成一份当前文章属性副本。
3. 请求当前飞书页面提取文章；成功后更新标题和正文预览。若页面脚本尚未注入，保留已生成的属性并提示刷新页面。
4. 用户编辑的是当前文章副本，不直接修改模板。
5. 用户进入模板设置页时，从已保存模板创建草稿；只有点击保存才持久化。
6. 点击主按钮后先完成属性、目录、页面类型和 Vault 权限校验。
7. 正文继续使用现有转换器转换为 Markdown，图片继续使用现有缓存、读取和写入流程。
8. 使用当前文章属性生成 YAML frontmatter，并与正文组合后交给现有 Vault 写入流程。
9. 保存完成后显示笔记相对路径和图片成功/失败数量。

`captured_at` 与 `source_type` 不再作为不可配置的隐藏属性强制写入。用户若需要，可在模板中自行添加相应字段，保证最终属性完全由模板和本次编辑控制。

## 模块边界

- `popup.html`：主编辑页、模板设置页和底部操作区的静态结构。
- `popup.css`：深色视觉、响应式高度、编辑控件和状态样式。
- `src/popup.js`：页面状态、事件绑定、Vault 流程、文章读取与保存编排。
- 新增属性领域模块：负责默认模板、实例化、校验、类型转换和 YAML 序列化；模块不依赖 Chrome API 或 DOM，可独立测试。
- `src/markdown.js`：继续负责正文 HTML 到 Markdown 的转换；调整为接收外部生成的 frontmatter，避免属性逻辑分散。
- `src/vault.js`：继续负责目录与文件写入和同名文件防覆盖，不感知属性编辑界面。

## 失败边界

以下情况禁止写入并显示明确错误：

- 当前标签页不是受支持的飞书 Wiki 或文档页面。
- Vault 未选择、权限被拒绝或权限恢复失败。
- 笔记目录或附件目录不是合法的 Vault 相对路径。
- 属性字段名为空、重复或包含控制字符。
- 日期或数字值不符合对应类型。

失败时保留当前全部编辑内容，不清空属性，不重复创建文件。字段错误需要在属性行显示并聚焦到对应字段。文章标题为空时，文件名使用“未命名笔记”；`title` 属性可改名、删除或停用，不影响这一文件名回退规则。

图片失败沿用现有降级策略：正文继续保存，失败图片保留远程链接，并在保存结果中显示警告。

## 验证方案

### 自动测试

- 默认模板结构与自动值解析。
- 当前文章属性与模板草稿互不污染。
- 五种属性类型的合法值、空值和非法值。
- 字段名为空、重复和包含控制字符的校验。
- 文本引号、换行、冒号等 YAML 特殊内容的安全序列化。
- 列表、日期、布尔值和数字的 YAML 类型输出。
- 属性排序和停用字段的输出行为。
- 原有正文转换、图片、路径和 Vault 写入测试继续通过。

### 浏览器人工验收

- 对照参考截图检查深色布局、字号、间距、滚动和底部固定区。
- 验证每种类型的新增、编辑、改名、删除、排序和切换类型。
- 验证模板保存后重新打开插件仍存在，取消模板修改不会持久化。
- 验证单篇临时修改不会改变模板默认值。
- 在真实飞书文档上验证正文预览、Vault 首次选择、权限恢复、图片保存和最终 Markdown。
- 在 Obsidian 中打开结果，确认 Properties 面板识别出正确类型。

## 非目标

- 第一版不支持多个命名模板之间切换；界面保留 `Default` 名称，为后续扩展留出位置。
- 不在弹窗中编辑 Markdown 正文。
- 不同步 Obsidian 已有的 Properties 类型定义。
- 不处理评论、修订历史、画板、数据库视图、电子表格或复杂第三方嵌入。
