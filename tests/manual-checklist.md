# 浏览器人工验收清单

## 准备

- [ ] 在 `chrome://extensions` 以“加载已解压的扩展程序”载入项目目录。
- [ ] Chrome 未显示 Manifest、CSP 或内容脚本错误。
- [ ] 准备一个不含重要数据的测试 Vault。
- [ ] 打开已登录且有权限的飞书 Wiki/文档，刷新页面。

## 授权与设置

- [ ] 首次弹窗显示“选择 Obsidian Vault”，系统选择器由按钮点击直接触发。
- [ ] 取消目录选择不会写文件，也不会显示成功。
- [ ] 选择测试 Vault 后显示 Vault 名称和“可写入”。
- [ ] 笔记目录留空、附件目录使用默认值时可以保存。
- [ ] 填入不存在的相对目录时自动创建目录。
- [ ] 输入 `../outside` 或绝对路径时拒绝保存并显示明确错误。
- [ ] 关闭并重新打开弹窗后设置仍保留。
- [ ] 浏览器要求重新授权时，“恢复 Vault 授权”可以恢复；失败时可重新选择目录。

## 正文与格式

- [ ] 文件名来自飞书标题，非法字符被清理。
- [ ] YAML 包含标题、原 URL、本地时区抓取时间和 `source_type: feishu`。
- [ ] 段落、标题、粗体、斜体、删除线和链接在 Obsidian 中正确显示。
- [ ] 有序列表、无序列表、嵌套列表和任务列表正确显示。
- [ ] 引用、行内代码、代码块及语言标记正确显示。
- [ ] 表格列数和内容正确，单元格中的 `|` 已转义。
- [ ] 页面导航、工具栏、目录和评论没有混入正文。

## 文件与图片

- [ ] 图片写入配置的附件目录，Markdown 使用 `![[相对路径]]`。
- [ ] 同一文档连续保存两次会生成 `标题.md` 和 `标题-2.md`，旧文件内容不变。
- [ ] 对应附件也使用不同文件名，旧图片不被覆盖。
- [ ] 图片请求失败时正文仍保存，失败图片保留远程 Markdown 链接，结果显示警告。

## 错误路径

- [ ] 在非飞书页面中按钮禁用，并提示当前页面不受支持。
- [ ] 未登录、无权限或文档不存在时不创建空笔记，并显示访问提示。
- [ ] 正文仍在加载或识别结果过短时不创建文件。
- [ ] 安装/重载插件后未刷新的旧标签页提示刷新页面。
- [ ] 撤销目录权限后不会宣称保存成功。
